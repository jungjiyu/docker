	: FTP , Talnet 과 같은 프로토콜 도 있는데 , 왜 SSH 를 사용하는가? >> 보안 떄문에.
		: FTP 와 Talnet 과 같은 프로토콜은 암호화 없이 네트워크를 통해 직접 데이터를 넘기기 떄문에, 누구나 해당 데이터를 열어볼 수 있어 보안에 상당히 취약하지만, SSH 의 경우 훨씬 안전한 채널을 구성 한 뒤(암호화한뒤) 정보를 교환하기 떄문에 보안적인 면에서 훨씬 뛰어남
	: 작동 방식 >> 다른 컴퓨터와 통신위해 접속하기 위한 인증을 (비밀번호가 아닌) 한쌍의 key 를 통해 진행한다
		: 한 쌍의 key >> private key, public key
			: public key >> 공개되어도 비교적 안전한 key
				: 암호화는 가능하지만, 복호화는 불가능
				: 이 키를 통해, 메시지를 전송하기 전 암호화를 하게 된다
			: private key >> 외부에 노출되어서는 안되는 key
				: 본인의 컴퓨터 내부에 저장하게 되어있다.
				: 이 키를 통해, 전송된 (암호화된) 메시지를 복호화 가능하다.

IP 주소 Internet Protocol address : 컴퓨터 네트워크 상에서 사용되는 컴퓨터 주소 
	: IP 주소를 IP 라고 줄여서 말하는 경우도 많은데, 이건 편의상 줄임말로 쓴거지 실제로 IP 자체는 Internet Protocol 즉 인터넷 규약을 말하는 것임 주의.
	: 종류
	    (1) 공인/사설
		1. 공인 ip (= 외부 ip) : 세상에서 유일한 ip 주소
		2. 사설 ip (= 내부 ip) : 내부 네트워크 상에서 각 컴퓨터 간 통신하기 위해 활용되는 ip 주소
			: 0~225까지의 숫자 4개로 구성되는 IPv4는 약 42억개의 주소만 할당할 수 있고,
이를 전세계 모든 장비에 할당하기에는 현재 아이피 체계가 많이 부족하다.
그래서 외부 네트워크가 필요하지 않은 장비에 대해서는 별도의 사설 네트워크를 구축하게 된다.
			:  보통 192.168.x.x 형태로 구성

	    (2) 고정/유동
		1. 고정 IP : 컴터꺼도 바뀌지 않는. 고정된 ip 주소
		2. 유동 IP. DHCP Dynamic Host Configuration Protocol : IP가 필요한 컴퓨터에 유동 IP를 부여하고 해당 컴퓨터를 종료하면 그 IP를 반환 받아서 다른 장치에 .다시 부여하는 것





NAT Network Address Translation >> IP 패킷의 TCP/UDP 포트 숫자와 소스 및 목적지의 IP 주소 등을 재기록(=다시 쓰면서. 변환하면서.)하면서 라우터를 통해 네트워크 트래픽을 주고 받는 기술
	:  내부 망에서는 사설 IP 주소를 사용하여 통신을 하고, 외부망과의 통신시에는 NAT를 거쳐 공인 IP 주소로 자동 변환된다.
	: 사용하는 이유>> 
		1. 사설 네트워크에 속한 여러 개의 호스트가 하나의 공인 IP 주소를 사용하여 인터넷에 접속하기 위함
		2. 사용자의 통신망 보호 목적.

			: 로그인 성공시 왼쪽에 {username}@{servername}~$ 과 같이 뜬다.
				: 일반사용자권한 로그인 의미

		2. net-tools 설치 >> sudo apt install net-tools
			: net-tools >> 네트워크 관련 기본적인 명령어를 사용할 수 있게 함
				: 리눅스 네트워크 서브 시스템을 사용하기 위한 툴이 기본적으로 설치되어 있지 않아서 별도로 이렇게 다운 받아줘야된다.
				: ifconfig, netstat 등 사용 가능하게 해준다.
				    : ifconfig >> 가상머신의 IP 주소 (첫 inet 키워드의 바로 뒤) 확인 가능.
					: "interface configuration"의 약자
					: ipconfig 가 아니라 ifconfig 임을 주의
					: 내경우는 10.0.2.5 였음
			: https://itlearningcenter.tistory.com/entry/%E3%80%90Ubuntu-2004-LTS%E3%80%91net-tools-%EC%84%A4%EC%B9%98%ED%95%98%EA%B8%B0

		3. openSSH 설치여부 확인 >> sudo systemctl status ssh
			: active 이라는 초록 글자가 보이면 정상 작동하고 있는 것.

	: 가상머신 종료 유형
		1. 현재 시스템 상태 저장(Save the Machine State) >> 실제 물리적 컴퓨터에서 "절전"과 동일
		2. 컴퓨터 끄지 신호 보내기  (Send the Shutdown Signal) >> 실제 물리적 컴퓨터에서 "시작 메뉴"를 통해 "종료"를 선택하는 것과 동일
			: 종료하는 데 시간이 걸릴 수 있다.

		3. 시스템 전원 끄기  (Power Off the Machine) >> 물리적 컴퓨터의 전원 버튼을 강제로 눌러 전원을 끄는 것과 동일. 가상 머신의 전원을 즉시 끄는 것. 
			: 빨리 종료시킬 수 있다.
			: 저장되지 않은 작업 내용이 손실될 수 있다.

	: 가상머신 전원 키기 >> "시작 ->" 버튼 누르기

	: 가상머신 네트워크 환경 설정하기
		1. virtual box 홈 화면에서, 왼쪽 큼지막한 "도구" 탭의 왼쪽 찍찍 아이콘 - 네트워크 클릭
			(1) 옆의 "NAT 네트워크" 탭 클릭
			(2) "만들기 + "아이콘 클릭
			(3) 일반옵션(G) 탭에서 DHCP 활성화 선택되있는지 확인

		2.  앞서 생성한 가상 머신 탭 클릭 후, 설정(톱니바퀴 아이콘) 클릭 
			(1) 어댑터1 에서 , "다음에 연결됨 ▼" 을 클릭해 NAT 네트워크 선택, 확인버튼 누름
				: 그냥 "NAT" 와 "NAT 네트워크"는 다른거임 주의
			
		3. 가상머신 전원킴(시작버튼 누름) 
			(1) ifconfig 로 현재 ip 주소 확인. (첫 inet 키워드 바로 옆에 뜨는거)
				
		4. ( 가상머신 전원 여전히 켜져있는 상태에서-PuTTY 설치를 위해선 가상머신이 가동중이여야됨 ) 다시 virtual box 홈 화면에서, 왼쪽 큼지막한 "도구" 탭의 왼쪽 찍찍 아이콘 -  네트워크 클릭
			(1) 옆의 "NAT 네트워크" 탭 클릭
			(2) 포트포워딩(P) 탭에서 옆의 조그만 + 아이콘 누르고, 생성되는 row 세부설정 변경
				1. 호스트 ip >> 빈칸으로 냅둔다
				    : 서버의 내부적인 ip. 사설 ip 역할.
				2. 호스트 포트 >> 22
				3. 게스트 ip >> 아까전에 ifconfig 로 확인해둔 ip 주소
				    : 클라이언트가 요청.접근하게 되는 ip. 공인 ip 역할.
				4. 게스트 포트 >>22

	: PuTTY 설치하기	
		: 옵션이긴 하지만 있으면 편하므로 다운 받는다.
			: 다운받으면 .msi 파일이 다운받아진다. 
			: 왠지는 모르겠는데 공식 사이트는 접근이 불가해서 softnic 인가 그 사이트에서 다운로드 받음
		: 설치 후 글꼴 설정을 한다(그냥 보긴 좀 가독성 떨어짐) 
			: window-appearance-change 에서 글꼴을 consolas, 크기는 11 로 한다

		: 시작하기 >>session 에서 host name 을 127.0.0.1 로 설정하고 open 클릭
			: 매번 host name 설정하기 귀찮으면 해당 설정을 save 해두자.
				: 저장해두면 그거 클릭하고 open 해도 되고, 더블클릭해도 된다.

		: 초기 설정하기 
			: 실습디렉터리 만들기 >> mkdir 폴더명
			: 현재 디렉토리 내의 파일, 폴더 목록 확인 >> ls
			: vim 편집기 사용해 텍스트 파일 생성 >> vim 파일명.txt
			: 생성한 텍스트 파일 편집 
				1. i 눌러서 INSERT 모드로 전환해 글자 입력할 수 있게 함
				2. 테스트용 내용 작성
				3. shift + ; 눌러서 COMMAND 모드로 전환해 명령어 입력할 수 있게 함
				4. wq 입력 후 엔터해 저장 및 편집기 종료
			: 생성한 텍스트 파일의 위치 변경 >> mv 파일명.txt 폴더명
